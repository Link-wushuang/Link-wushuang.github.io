Hugo-Reimu主题美化：随机横幅、封面轮换与图片裁剪配置

这篇文章记录一下我对博客做的几个图片相关的优化，主要包括：

  • 顶部横幅每次刷新随机切换
  • 首页文章封面图按顺序轮换（跨分页保持连续）
  • 每张图片单独设置裁剪位置，避免截掉人脸
  • 修复朋友页头像不显示的问题

一、顶部横幅随机切换

Reimu主题的横幅图是在 params.yml 里面配置的，是静态的，每次打开都是同一张。想要随机的话需要用 JS 在页面加载时动态替换。

1. 准备横幅图片

把你想用的横幅图片放到 static/images/ 目录下。

2. 在 afterFooter.html 添加 JS

打开 layouts/partials/afterFooter.html，在文件末尾添加以下代码：

<script data-pjax>
(function () {
  var banners = [
    { src: '/images/banner1.jpg',  pos: '50% 40%' },
    { src: '/images/banner2.jpg',  pos: '50% 40%' },
    { src: '/images/marin1.jpg',   pos: '50% 8%'  },
    { src: '/images/marin2.jpg',   pos: '50% 15%' },
    // 继续添加你的图片...
  ];

  function applyBanner() {
    var b = banners[Math.floor(Math.random() * banners.length)];
    var picture = document.querySelector('#page-header picture');
    var img     = document.querySelector('#page-header img');
    if (picture) {
      var sources = picture.querySelectorAll('source');
      sources.forEach(function (s) { s.srcset = b.src; });
    }
    if (img) {
      img.src = b.src;
      img.style.objectPosition = b.pos;
    }
  }

  if (document.readyState === 'loading') {
    document.addEventListener('DOMContentLoaded', applyBanner);
  } else {
    applyBanner();
  }
})();
</script>

每个图片对象里面的 pos 就是裁剪位置（下面会详细说），这里先填 50% 40% 就行。

data-pjax 这个属性是为了让使用了 pjax 的主题在页面跳转时也能重新执行这段脚本。

二、首页封面图顺序轮换

默认情况下，Reimu 主题用 randomCover.html 这个 partial 来给每篇文章分配封面。不过这个方式有个问题：Hugo 构建时是并发渲染的，导致分配顺序不稳定，看起来"随机"但其实没规律。

想要 1→2→3→…→11→1 这样按顺序循环的话，改用索引取模的方式更可靠。

1. 配置 data/covers.yml

在 data/covers.yml 里面按顺序列出所有封面图：

- images/marin1.jpg
- images/marin2.jpg
- images/marin3.jpg
- images/marin4.jpg
- images/marin5.jpg
- images/marin6.jpg
- images/marin7.jpg
- images/marin8.jpg
- images/marin9.jpg
- images/marin10.jpg
- images/marin11.jpg

2. 修改 post.html

打开 layouts/partials/post.html，找到没有自定义封面时的那个 else 分支，把 randomCover.html 的调用替换成按索引取封面：

{{- else -}}
  {{- $covers := $ctx.Site.Data.covers -}}
  {{- $cover := index $covers (mod $index (len $covers)) -}}
  <img
    data-src="{{ $cover | relURL }}"
    data-sizes="auto"
    alt="{{ $page.Title }}"
    class="lazyload"
  />
{{- end -}}

这里的 $index 就是文章的全局序号，用 mod 对封面图数量取余，就能实现循环。

3. 修改 index.html 计算全局偏移

分页的时候，每一页的文章序号 $k 都是从 0 开始的，所以第 2 页的第 1 篇文章序号还是 0，封面图就跟第 1 页的第 1 篇一样了。

需要在 layouts/index.html 里面算出当前页之前有多少篇文章，然后加进去：

{{- $offset := 0 -}}
{{- range $paginator.Pagers -}}
  {{- if lt .PageNumber $paginator.PageNumber -}}
    {{- $offset = add $offset (len .Pages) -}}
  {{- end -}}
{{- end -}}

然后把所有调用 post.html 的地方，index 参数从 $k 改成 (add $offset $k)，比如：

{{- partial "post.html" (dict "index" (add $offset $k) "ctx" $ "page" $v) -}}

这样第 2 页的文章就会从第 1 页的末尾继续算，封面图也就接着轮换了。

三、底部导航图片同步封面顺序

文章页底部的上一篇/下一篇导航也有缩略图。想让这个缩略图和该文章在首页显示的封面一致的话，需要找到这篇文章在全站的全局序号，然后用同样的 mod 逻辑取封面。

打开 layouts/partials/post/nav.html，找到没有自定义封面时的图片部分，改成这样：

{{- $covers := .Site.Data.covers -}}
{{- $allPages := where .Site.RegularPages "Section" "in" .Site.Params.mainSections -}}

{{- if .PrevInSection -}}
  {{- $prevIdx := 0 -}}
  {{- range $i, $p := $allPages -}}
    {{- if eq $p $.PrevInSection -}}{{- $prevIdx = $i -}}{{- end -}}
  {{- end -}}
  {{- if and .PrevInSection.Params.cover (not (strings.HasPrefix .PrevInSection.Params.cover "rgb")) -}}
    <img data-src="{{ .PrevInSection.Params.cover | relURL }}" ... />
  {{- else -}}
    <img data-src="{{ index $covers (mod $prevIdx (len $covers)) | relURL }}" ... />
  {{- end -}}
{{- end -}}

下一篇同理，用 $nextIdx 做一样的操作。

四、图片裁剪位置控制

封面图和导航图用的是 object-fit: cover，默认会居中裁剪。如果图片里人物的脸在偏上或偏下的位置，就需要调整 object-position。

用 lazysizes 的 lazyloaded 事件来动态设置，在 afterFooter.html 里面添加：

var coverPositions = {
  '/images/marin1.jpg': '50% 8%',
  '/images/marin2.jpg': '50% 8%',
  '/images/marin5.jpg': '50% 20%',
  '/images/marin7.jpg': '50% 8%',
  // 需要调整的图片都加进来
};

document.addEventListener('lazyloaded', function (e) {
  var img = e.target;
  var src = img.getAttribute('src') || '';
  var key = src.replace(/^.*?(\/images\/)/, '/images/');
  var isNav = !!img.closest('#article-nav');
  if (isNav) {
    img.style.objectPosition = '50% 25%';  // 导航图统一用这个
  } else if (coverPositions[key]) {
    img.style.objectPosition = coverPositions[key];
  }
});

object-position 的格式是 水平位置 垂直位置，垂直方向上数字越小越往上，越大越往下。比如 50% 8% 就是靠上显示，适合人物脸在图片上方的情况。

导航图和首页卡片的区分方式是 img.closest('#article-nav')，在导航区域内的就是导航图，否则就是首页封面卡片。

五、修复朋友页头像不显示

如果你的朋友页头像一直不显示（透明的），大概率是 friendsLink.html 里面用的是普通 src 而不是懒加载的 data-src。

Reimu 主题的友链样式里 .friend-avatar 默认是 opacity: 0，只有加了 lazyloaded 这个 class 之后才变成 opacity: 1。如果不走懒加载，这个 class 永远不会加上去，头像就一直是透明的。

打开 layouts/shortcodes/friendsLink.html，找到头像 img 标签，改成懒加载的写法：

<img
  class="friend-avatar lazyload no-lightbox"
  data-src="{{ .image }}"
  alt="{{ .name }}"
/>

把 src 改成 data-src，然后在 class 里面加上 lazyload，就好了。

小结

这几个改动主要是：

  1. JS 随机横幅 + data-pjax 支持 pjax 跳转
  2. 封面图用索引取模代替 Scratch 队列，解决并发渲染的不稳定问题
  3. 全局偏移保证分页时封面连续
  4. lazyloaded 事件动态设置裁剪位置
  5. 友链头像改为懒加载写法

都是一些小改动，但折腾起来还是花了不少时间，希望对也在用 Reimu 主题的朋友有帮助嘻嘻
